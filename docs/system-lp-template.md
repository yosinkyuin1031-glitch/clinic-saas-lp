# 個別LP（/systems/[id]）共通テンプレート

ClinicMark配下のすべての個別LPで使う共通構造。基準は `app/systems/customer/page.tsx`（Clinic Core）。

---

## 全体方針

- アプリごとに濃淡を変えるが、**セクション順は固定**
- 「必須」セクションは全アプリで揃える
- 「任意」セクションはアプリの性質に合わせて入れる／抜く
- 業種訴求は **整体・鍼灸・接骨院・サロン** の4軸で統一
- 治療家としての本業（治療＋スタッフ教育）の時間を取り戻す、というメッセージ軸を全アプリで通す

### 任意セクション（該当アプリのみ）

| セクション | 入れる条件 |
|---|---|
| モニター枠の限定訴求 | モニター募集中（Clinic Coreなど） |
| 開発ストーリー（なぜ作ったか） | 看板アプリ（カラダマップ・Clinic Core・MEO） |
| 他システムとの連携 | 連携可能なもの（Clinic Core・WEB問診・予約管理など） |
| 他サービスとの比較表 | 大手競合があるもの（MEO・顧客管理・予約など） |
| AI訴求 | AI機能を実装しているもの（Clinic Core・カラダマップなど） |
| セキュリティ | 患者データを扱うもの（Clinic Core・WEB問診・カラダマップなど） |
| 導入事例 | 実績が出てきたら（テンプレ完成後フェーズ2で投入） |

## 業種別ユースケースの書き方（★新セクション）

各アプリで以下の4軸を埋める：

```
【整体院での使い方】
（例：問診→検査→施術提案→PDF配布までを1分短縮）

【鍼灸院での使い方】
（例：施術後の経過追跡を自動化）

【接骨院での使い方】
（例：保険請求のためのカルテ作成を効率化）

【サロンでの使い方】
（例：常連客の体調変化を可視化して提案につなげる）
```

無理に4業種すべて埋めず、当てはまらない業種は「△」表示で簡素にする。

---

## 文言の統一ルール

- 「治療家」「施術家」を業種に応じて使い分け（サロンは「施術家」）
- 業種列挙の順は **整体・鍼灸・接骨院・サロン** で固定
- 「整骨院」は使わず **接骨院** で統一
- 「自律神経」を業種として書かない（治療法・症状名としてのみOK）
- 院名表記：大口神経整体院（正式名）
- 関連事業：晴陽鍼灸院（訪問鍼灸リハビリ）
